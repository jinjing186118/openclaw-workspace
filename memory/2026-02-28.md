## 🔄 Mac 本地开发 → 腾讯云同步机制（最终方案）

### 架构
```
MacBook 本地开发                    腾讯云服务器
┌─────────────────┐                ┌─────────────────┐
│ ~/openclaw-     │                │ ~/.openclaw/    │
│ workspace/      │ ──git push──►  │ workspace/      │
│                 │                │ (OpenClaw运行)  │
│ Codex/Claude    │ ◄──git pull────│                 │
│ Code 辅助开发   │                │ Gateway + Bot   │
└─────────────────┘                └─────────────────┘
```

### 工作原理
1. **代码位置统一**：GitHub 仓库作为唯一数据源
2. **本地开发**：MacBook 克隆仓库，用 Codex/Claude Code 辅助编辑
3. **版本控制**：Git 记录所有修改，支持回滚和协作
4. **云端部署**：腾讯云通过 git pull 获取最新代码，deploy.sh 重启生效

### 操作流程

#### 1. 克隆仓库（一次性）
```bash
cd ~
git clone https://github.com/jinjing186118/openclaw-workspace.git
```

#### 2. 日常开发
```bash
cd ~/openclaw-workspace

# 拉取最新代码
git pull origin main

# 用 AI 工具打开（推荐）
codex .
# 或 claude .
```

#### 3. 提交修改
**方式A - AI 命令（推荐）：**
```
帮我把修改提交，message 写"feat: xxx"
```

**方式B - 终端命令：**
```bash
git add .
git commit -m "feat: xxx"
git push origin main
```

**方式C - VS Code 图形界面：**
左侧 Git 图标 → 输入 message → Commit → Push

#### 4. 部署到腾讯云
```bash
ssh root@43.165.2.205 "cd ~/.openclaw/workspace && git pull && ./deploy.sh"
```

### 与 SSH Remote 开发的对比

| 方式 | Git 本地编辑 | VS Code Remote SSH |
|------|-------------|-------------------|
| 文件位置 | Mac 本地有一份 | 只在腾讯云 |
| 编辑体验 | 本地 VS Code（快）| 远程 VS Code（依赖网络）|
| 版本控制 | ✅ 有 Git 历史 | ❌ 直接改，无历史 |
| 多人协作 | ✅ 不会冲突 | ⚠️ 可能覆盖他人修改 |
| 离线可用 | ✅ 可以 | ❌ 必须联网 |
| 生效速度 | push → pull → restart | 保存即生效 |
| 适合场景 | 批量修改、新功能开发 | 紧急小修、快速调试 |

### 推荐组合
- **主力开发**：Git + Codex/Claude Code（有版本历史，AI 辅助）
- **紧急修复**：SSH Remote（快速，无需 push/pull）

### 注意事项
- CLI 远程连接配置失败（OpenClaw 版本不支持），但不影响整体方案
- 所有敏感数据（API keys、数据库）留在腾讯云，不进入 Git
- 每次部署前确保已 commit 并 push
