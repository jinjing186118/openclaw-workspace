# OpenClaw 团队架构 & 通信机制

> 金主人备注：后面可能需要分享给别人（2026-03-01）

## 1. 飞书入口路由（用户 → Bot）

每个bot绑定一个独立的飞书应用，发消息给谁就进谁的会话：

```
用户的飞书
  ├─ 发给「金金的管家」    → main（大主管）
  ├─ 发给「开发助手」      → dev
  ├─ 发给「材料撰写助手」  → content
  ├─ 发给「热点搜集助手」  → ops
  ├─ 发给「视频助手」      → vedio
  ├─ 发给「公众号IP写作」  → brand
  └─ 发给「需求助手」      → requirements
```

## 2. Bot之间的通信方式

```
┌─────────────────────────────────────┐
│          sessions_send()            │
│       (Bot间直接发消息通信)          │
│                                     │
│   main ◄──────► dev                │
│    ▲ ◄──────► content              │
│    │ ◄──────► ops                  │
│    │ ◄──────► vedio                │
│    │ ◄──────► brand                │
│    │ ◄──────► requirements         │
│    │                                │
│    └─ 大主管可以调度所有人           │
│       其他bot之间也可以互相通信      │
└─────────────────────────────────────┘
```

通信代码示例：
```json
// main 给 ops 发任务
sessions_send({
  "agentId": "ops",
  "message": "帮我搜今天的AI热点"
})

// ops 给 brand 同步素材
sessions_send({
  "agentId": "brand",
  "message": "这条热点适合写公众号..."
})
```

## 3. 定时任务流程（以AI早报为例）

```
⏰ 每天9:00 (cron触发)
    │
    ▼
🎩 main（大主管）收到任务
    │
    ▼ sessions_send → ops
🔥 ops（热点搜集助手）执行搜索
    │
    ▼ 搜索完成，结果返回
🎩 main 收到结果
    │
    ▼ 推送到飞书
📱 用户看到早报
```

## 4. 当前配置总览

| 维度 | 配置 |
|------|------|
| 模型 | 全员 anthropic/claude-opus-4-6 |
| 通信协议 | agentToAgent: enabled |
| 允许互通 | 全部7个bot互相可通信 |
| 入口 | 每个bot独立飞书应用 |
| 工作区 | 每个bot独立目录（隔离） |
| 记忆 | 各自独立，通过文件同步共享知识 |

## 5. 关键特点

- **会话隔离：** 每个bot有自己的记忆和上下文，不会互相干扰
- **主动协作：** 通过 sessions_send 主动找对方，不是共享大脑
- **大主管统筹：** main 可以调度任何人，也是定时任务的调度中心
- **结果回传：** 子任务完成后结果会回到发起者

## 6. 各Bot底层区别

**硬件层完全一样**（同一个Claude模型、同样的工具和技能）

**区别在软件层：**
| 文件 | 作用 |
|------|------|
| SOUL.md | 性格、思维框架、工作SOP（每个bot不同） |
| USER.md | 对用户的了解侧重点（每个bot定制） |
| MEMORY.md | 记忆积累（各自独立） |
| memory/*.md | 共享知识文件（已同步） |

**main的特殊之处：** 设为default bot + 定时任务调度中心 + 统筹权限
