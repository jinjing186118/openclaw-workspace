# 公众号运营方法论（洋哥课程体系）

> 来源：洋哥《公众号基础入门》系列课程（2026年2月）
> 整理：金金的CEO 🎩

---

## 📌 核心心法（贯穿全部课程）

1. **标题定江山** — 标题占80-90%权重，没有好标题就不要写
2. **故事化写作 + 写作带情绪** — 公众号最关键的两个能力
3. **加微才是王道** — 考核指标不是阅读量，是加微信数（私域引流）
4. **素材一鱼多吃** — 朋友圈→公众号→短视频→直播，内容循环利用
5. **真诚利他** — 不说谎、不编故事、不虚假夸大
6. **AI是枪，你是扳机** — AI写大部分，故事和情绪必须你亲自主刀

## 第一课：公众号运营入门核心指南

### 为什么还要做公众号
- **变现最佳阵地**：中小IP的最佳变现平台，成交率高
- **信任放大器**：半公域半私域，最接近个人微信
- **内容沉淀**：用户通过历史文章了解你的思想体系

### 适合做公众号的人
1. 内容型个人IP — 打造长期内容资产
2. 知识/服务产品提供者 — 建立专业形象
3. 依赖私域成交的业务 — 增强客户信任

### 三个关键打通动作（公众号+视频号）
1. 公众号视频号一体号（直播关注的是公众号）
2. 视频号简介露出公众号
3. 直播/评论区置顶导流

### 账号设置核心
- **注册类型**：个人IP选订阅号，企业选服务号
- **命名黄金法则**：好记（≤7字）+ 含关键词 + 可长期使用 + 全网一致
- **命名范式**：昵称 + 细分行业（如：杨哥聊AI商业）

### 头像设计
- 一眼识别你是谁，与定位匹配
- 清晰大于好看
- 长期一致，小幅迭代

### 简介设计
- 我是谁？帮谁解决什么问题？
- 高光时刻 or 反差故事
- 3秒内让用户看懂，标签≤3个

### 起号三件套
1. 明确我是谁/解决什么问题/给谁看
2. 万能选题公式：经历转变反思、问题方法结果、拆别人说自己
3. 定发布节奏：每周2-3篇，固定时间（8/12/18/21点）

### 菜单栏设置
- 三大作用：行为引导、内容聚合、提升搜索曝光
- 结构：必读（IP文）+ 精华内容 + 联系进群领资料

## 第二课：公众号爆款标题方法论

### 爆款标题公式 ⭐⭐⭐
**爆款标题 = 流量关键词 × 冲突悬念 × 情绪钩子 × 数字 × 动词**

不需要5个全有，2-3个就够。但标题越长，需要的要素越多。

### 流量关键词三类
1. **当下热点**：DeepSeek、元宝红包、董宇辉等
2. **固有大名词**：清华/北大/985/腾讯/阿里等
3. **情绪类词**：涨薪了、失业了、被裁了、离职了

### 不同平台的流量关键词
| 平台 | 热门关键词 |
|------|-----------|
| 公众号 | 离职、裁员、小镇做题家、阶层跨越 |
| 知乎 | 认知偏差、信息茧房、幸存者偏差 |
| 小红书 | 微胖女孩、通勤穿搭、极简、100天挑战 |
| 教育号 | 学区房、高考、鸡娃 |
| 情感号 | 彩礼、离婚、婆媳 |

### 冲突悬念三招
1. **反常识**：越懒的人越能赚钱、越省钱的人越穷
2. **场景化**：凌晨3点加班改PPT，我才明白应该早点离职
3. **反转**：被公司开除后，我赚到了人生第一桶金

### 情绪钩子分类
- 人性弱点：贪婪、恐惧、懒惰
- 痛点挖掘：涨薪了、失业了
- 行业揭秘：健身教练都不敢说的减肥方法
- 惊喜体验：极致赞美、感官冲击
- 紧迫行动：最后3天、限时

### 数字运用
- **对比制造冲突**：从月薪5000到3万，我用了3个月
- **增强可信性**：从业10年的编辑告诉你
- **证明简单易操作**：3分钟学会AI写PPT

### 标题天条
- ❌ 不要反问读者"如何""为什么"（读者没粉你之前觉得自己是你爹）
- ❌ 不要生僻字
- ✅ 控制长度，手机端≤20字，关键信息前置
- ✅ 准备3-5个标题做AB测

### 标题优化工作流
1. **热词采集**：微博热搜、知乎热榜、微信指数、AI工具
2. **冲突设计**：结合热词制造冲突点
3. **AB测试**：人肉测（粉丝/亲友投票）
4. **效果评估**：分析打开率、阅读量、转发量

### 垂直领域标题模板
- **工具评测类**：人群 + 痛点场景 + 工具数量 + 效果承诺
  - 例：跨境电商卖家必看，这3个AI选品工具让爆款率提升200%
- **技术解读类**：反常识结论 + 技术名词 + 生活化类比
- **副业变现类**：时间紧迫性 + 收益证明 + 操作简易度
- **行业预测类**：时间节点 + 颠覆性预测 + 行动指令

### 爆款标题 vs 标题党
- 爆款标题：标题藏一手，但内容对应标题（"团队准备解散了"→朋友的团队）
- 标题党：标题和内容完全无关（"一个时代结束了"→赶紧买黄金）

## 第三课：一篇标准的IP文写作指南

### IP文的本质
- 不是简历流水账，不是心灵鸡汤
- 是**个人商业故事** — 人生起承转合、高光低谷
- AI时代核心竞争力：真实的故事 + 独特的情绪

### IP文六大价值
1. **筛选精准粉丝**：吸引同频人群
2. **建立信任**：转化的基础
3. **塑造差异化人设**：在众多创作者中脱颖而出
4. **沉淀个人品牌资产**：永久的内容基石和流量入口
5. **促进商业转化**：信任建立后推荐水到渠成
6. **痛点共鸣**：与读者产生共振

### IP文万能框架 ⭐⭐⭐
```
困境/低谷/痛点 
→ 觉醒/转折点 
→ 探索/奋斗里程 
→ 突破/成就事件 
→ 核心方法论/价值观提炼 
→ 现状/愿景呼唤
```

**灵活运用**：
- 没有突破/方法论？去掉中间三步 → 困境 + 觉醒 + 现状 + 愿景
- 人生跌宕起伏？循环使用（多次困境→觉醒→突破）
- 现在很惨？写苦难 + 远大志向 = 成长型IP

### 核心写作法则
- **反差才是核心**：上帝给了烂牌，打成了王炸
- 现状越差 + 志向越大 = 能量值越强 = 穿透性越强
- **内容传播黄金法则**：口语化 + 情绪 + 故事 + 干货

### 情绪输出技巧
- 痛苦要场景化：不说"很穷"，说"连五块钱都拿不出来"
- 多用口语化：哇塞、我操、太厉害了
- 多用动词感官描写，少用形容词堆砌
- 结合具体场景和关键对话增强感染力

### 差异化定位
- 结合身份（宝妈/程序员）+ 人格 + 专长
- 突出核心能力与他人不同之处
- 例："宝妈在AI时代突破实现财务自由"

### IP文长度
- 首次：1500-3000字黄金区间
- 信息密度要高，避免注水
- 每3个月更新一次，随成长拔高

### IP文使用方式
- 置顶文章
- 关注回复/关键词回复推送
- 拆解成短视频、直播、朋友圈素材反复使用
- 加强读者对你的认知

### 避坑指南
1. ❌ 虚假夸大
2. ❌ 只有结果没有过程
3. ❌ 只有卖惨没有成长
4. ❌ 逻辑混乱（先画人生曲线梳理）
5. ❌ 自嗨式表达（用户用脚投票才是真理）
6. ❌ 没有召唤行动

### 行动清单
1. 挖掘5-10个关键事件（成就、低谷、转折点）
2. 明确定位：我是谁，帮谁解决什么问题
3. 选定最能体现定位的故事
4. 套用黄金框架填充大纲
5. 注入情绪细节
6. 写出初稿（垃圾初稿 > 完美幻想）

## 第四课：公众号增长策略

### 启动期：强力执行与精准触达
- 多渠道分发：改编后分发至微信群、社区、论坛
- 朋友圈：善用标签分组（职场内容对同事/领导不可见）
- 主动链接同行群：以提供价值的姿态参与互动
- 主动私信行业KOL寻求合作

### 成长期：白名单与互推
- **单开白**：为单篇授权转载 → 直接曝光引流
- **双开白**：允许修改不显示来源 → 深度合作、品牌渗透
- **互推**：与粉丝量级相当的号交叉增长

### 扩张期：跨平台引流
- 核心观点布局到其他内容平台
- 形成"外部平台拉新 → 公众号沉淀 → 用户参与扩散"闭环

### 一体化增长模型 ⭐
```
公众号 → (钩子) → 私域 → 社群 → 直播预约 → 直播 → (一体号涨粉) → 公众号
```
+ 短视频辅助爆发

## 第五课：公众号细分小号打造

### 核心观点
- 微信生态奖励垂直专业的人（2025微信公开课张小龙原话）
- **越细分越有精准流量，越精准越能变现**
- 专业 = 60分 + 实战（不需要满分）

### 细分定位层级示例
```
AI科普博主（太宽）
→ AI编程博主（可以）
→ AI编程·Cursor专精（推荐）
→ AI编程·产品出海（推荐）
```

### 真需求公式
**真需求 = 细分行业 + 具体场景 + 需求**

"饿了"是假需求（不知道吃什么）
"200斤想减肥吃什么"是真需求（细分场景+具体需求）

### 定位注意事项
- ✅ 可持续输出（一年100-150篇）
- ✅ 能蹭到风口热点
- ❌ 不要定位成产品（AI可研报告 → 教程写完就没了）
- ❌ 不要定位太宽泛（AI职场提效 → 给谁提效？）
- ❌ 不要定位成项目/潮流（AI艺人公司）

### 新号 vs 老号
- 新号有腾讯潜在红利，不比老号差
- 超过半年的杂乱老号 → 建议注销重开
- 三个月内的新号 → 可删旧文重新定位

### 热点结合定位
- 能结合 → 强转化/弱转化（AI视频号写Sora、AI编程号写Cursor做视频课）
- 不能结合 → 先吃热点流量再说（60-70%写垂直定位即可）
