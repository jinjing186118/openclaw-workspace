# AI Daily Digest 项目学习总结

> 来源：https://github.com/vigorX777/ai-daily-digest
> 学习日期：2026-03-01
> 目的：优化我们的热点抓取方法

## 🏗️ 核心架构：五步流水线

```
RSS抓取 → 时间过滤 → AI评分+分类 → AI摘要+翻译 → 趋势总结
```

这是最大的启发：**不是抓完就推，而是用AI做多层筛选和加工。**

## 🔑 关键技术点

### 1. 高质量信息源（90个Karpathy推荐博客）
- 不用搜索引擎，直接订阅RSS
- 信息源本身就是高质量的（HN热门博客）
- 覆盖：AI/ML、安全、工程、工具、观点

### 2. 并发抓取 + 容错
- 10路并发，15秒超时
- Promise.allSettled（不会因为一个失败全挂）
- 失败静默跳过，不影响整体
- 兼容RSS 2.0和Atom两种格式

### 3. AI三维评分体系（最值得学的）
- **相关性**（1-10）：对目标读者有多大价值
- **质量**（1-10）：文章本身的深度
- **时效性**（1-10）：现在读是否有意义
- 三维加权排序，Top N入选
- 同时完成：分类 + 关键词提取

### 4. AI摘要生成（结构化）
- 不是一句话敷衍，是4-6句结构化摘要
- 核心问题（1句）→ 关键论点（2-3句）→ 结论（1句）
- 额外生成：中文标题翻译 + 推荐理由
- 目标：30秒判断是否值得花10分钟读原文

### 5. 批量处理 + 降级机制
- 每批10篇文章一起发给AI评分（省token）
- 最多2个并发AI请求（避免限流）
- Gemini优先 → OpenAI兼容API降级
- JSON格式严格约束，解析失败有兜底分数

### 6. 可视化输出
- Mermaid饼图（分类分布）
- Mermaid柱状图（高频关键词）
- ASCII纯文本图（终端友好）
- 话题标签云

## 💡 我们可以借鉴的优化点

### 对ops热点搜集的改进
1. **从"搜索"升级到"订阅+搜索"双轨**
   - 固定RSS源保证基础覆盖
   - 搜索引擎补充突发热点
   
2. **引入AI评分机制**
   - 不是所有抓到的都推，要过评分筛选
   - 三维评分：与金主人相关性 + 质量 + 时效性

3. **结构化摘要**
   - 不只给标题和链接，要给30秒能看完的摘要
   - 附带推荐理由

4. **批量处理降低成本**
   - 多篇文章一次发给AI评分，而不是一篇一篇
   
5. **可视化统计**
   - 加入分类分布和关键词趋势
